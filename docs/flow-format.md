# VNE Flow Format Guide

This document describes the `.flow` file format used by VoidNovelEngine. Understanding this format helps you inspect, debug, and manually edit the flow files generated by `vne assemble`.

## Top-Level Structure

A `.flow` file is a UTF-8 encoded JSON object:

```json
{
  "max_uid": 42,
  "node_pool": [ ... ],
  "link_pool": [ ... ],
  "is_open": true
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `max_uid` | integer | Yes | Highest ID assigned to any entity in this flow. The next entity gets `max_uid + 1`. |
| `node_pool` | array | Yes | Array of node objects. Order does not matter. |
| `link_pool` | array | Yes | Array of link objects connecting pins between nodes. Order does not matter. |
| `is_open` | boolean | No | Whether this flow tab is open in the editor. Defaults to `true`. |

## ID System

All IDs (nodes, pins, links) share a single monotonically increasing counter per flow file.

When a node is created, IDs are assigned in this order:

```
node.id       = next_id()    // e.g., 1
input_pin[0]  = next_id()    // e.g., 2
input_pin[1]  = next_id()    // e.g., 3
output_pin[0] = next_id()    // e.g., 4
output_pin[1] = next_id()    // e.g., 5
```

When a link is created, it consumes one ID. `max_uid` must always equal the highest assigned ID.

IDs are never reused -- deleted entities leave gaps.

## Node Format

Every node has this structure:

```json
{
  "id": 1,
  "type_id": "entry",
  "position": { "x": 0, "y": 0 },
  "input_pin_list": [ ... ],
  "output_pin_list": [ ... ]
}
```

| Field | Type | Description |
|---|---|---|
| `id` | integer | Unique node ID |
| `type_id` | string | Node type (see node reference below) |
| `position` | `{x, y}` | Canvas position in integer pixels |
| `input_pin_list` | array | Ordered array of input pin objects |
| `output_pin_list` | array | Ordered array of output pin objects |

The `comment` node type has additional fields: `text` (string) and `size` (`{x, y}` object).

## Pin Format

```json
{
  "id": 2,
  "type_id": "flow",
  "is_output": false,
  "name": "Label",
  "val": "some value"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | integer | Yes | Unique pin ID |
| `type_id` | string | Yes | Pin type (see pin types below) |
| `is_output` | boolean | Yes | `true` for output pins, `false` for input pins |
| `name` | string | No | Display label. Omitted if the pin has no label. |
| `val` | varies | No | Stored value. Omitted for `flow` and `object` pins. |

### Pin Types

| Type | Category | Value Format | Example |
|---|---|---|---|
| `flow` | Control | No value | Execution flow connections |
| `object` | Data | No value | Runtime object references |
| `string` | Value | `"text"` | `"Hello world"` |
| `int` | Value | integer | `42` |
| `float` | Value | number | `1.5` |
| `bool` | Value | boolean | `true` |
| `vector2` | Value | `{x, y}` | `{"x": 100.0, "y": 200.0}` |
| `color` | Value | `{r, g, b, a}` | `{"r": 0.95, "g": 0.95, "b": 0.95, "a": 1.0}` |
| `texture` | Asset | string (asset ID) | `"study"` |
| `audio` | Asset | string (asset ID) | `"bgm_01"` |
| `video` | Asset | string (asset ID) | `"intro_video"` |
| `font` | Asset | string (asset ID) | `"default_font"` |
| `shader` | Asset | string (asset ID) | `"blur_shader"` |

Note: Color values use normalized floats (0.0 to 1.0), not 0-255 integers.

## Link Format

```json
{
  "id": 100,
  "input_pin_id": 4,
  "output_pin_id": 7
}
```

**Important -- counterintuitive field naming**: In the VNE format, the link field names describe the link's perspective, not the pin's perspective:

- `input_pin_id` = the pin feeding data **into** the link = a source node's **output** pin
- `output_pin_id` = the pin receiving data **out of** the link = a destination node's **input** pin

This means: data flows from `input_pin_id` to `output_pin_id`. The naming is the opposite of what you might expect.

### Connection Rules

- Connections go from an output pin (`is_output: true`) to an input pin (`is_output: false`)
- `flow` pins only connect to `flow` pins
- Same-type data pins connect to each other (e.g., `int` to `int`)
- `object` pins accept any non-flow type (wildcard)
- `int` output can connect to `float` input
- Each pin can have at most one connection

## Node Type Reference

### Control Flow

| Type ID | Description | Inputs | Outputs |
|---|---|---|---|
| `entry` | Flow starting point (one per flow) | -- | flow |
| `branch` | Conditional branch | flow, bool | flow (true), flow (false) |
| `loop` | Repeating execution | flow, flow (re-enter), flow (break), int (count) | flow (exit), flow (body), int (current) |
| `switch_scene` | Jump to another flow file | flow, string (scene ID) | -- |
| `switch_to_game_scene` | Jump to custom Lua scene | flow, string (module path) | flow |

### Presentation

| Type ID | Description | Key Inputs | Outputs |
|---|---|---|---|
| `delay` | Pause for N seconds | flow, float (seconds) | flow |
| `wait_interaction` | Wait for click/spacebar | flow, bool (enabled) | flow |
| `switch_background` | Change background image | flow, texture, float (fade), bool (wait) | flow |
| `add_foreground` | Add character/sprite | flow, float (scale), vector2 (position), texture, float (fade), bool (wait) | flow, object (reference) |
| `remove_foreground` | Remove character/sprite | flow, object (reference), float (fade), bool (wait) | flow |
| `move_foreground` | Animate sprite movement | flow, object, vector2 (target), float (duration), bool (wait) | flow |
| `show_letterboxing` | Show cinematic bars | flow, float (height), float (ease-in), bool (wait) | flow |
| `hide_letterboxing` | Hide cinematic bars | flow, float (ease-out), bool (wait) | flow |
| `show_subtitle` | Display subtitle text | flow, string (text), float (speed), float (bottom distance), font, int (size), color, bool (wait) | flow |
| `hide_subtitle` | Hide subtitle | flow | flow |
| `show_dialog_box` | Show dialogue box | flow, string (character), string (text), vector2 (position), float (width), float (fade), font x2, int x2, color x3, bool (wait) | flow, object (reference) |
| `hide_dialog_box` | Hide dialogue box | flow, object (reference), float (fade), bool (wait) | flow |
| `transition_fade_in` | Fade in from black | flow, float (duration), bool (wait) | flow |
| `transition_fade_out` | Fade to black | flow, float (duration), bool (wait) | flow |
| `show_choice_button` | Show up to 5 choice buttons | flow, string x5 (choice texts), font, int (size), color x4, int (spacing), vector2 (padding), float (bottom distance), float (min width) | flow x5 (branch routes) |
| `play_video` | Play fullscreen video | flow, video, int (fps), vector2 (resolution), float (volume) | flow |

### Audio

| Type ID | Description | Key Inputs | Outputs |
|---|---|---|---|
| `play_audio` | Play audio file | flow, audio, int (loops), float (volume), float (fade-in) | flow, int (channel) |
| `stop_audio` | Stop specific channel | flow, int (channel), float (fade-out) | flow |
| `stop_all_audio` | Stop all audio | flow, float (fade-out) | flow |

### Variables

| Type ID | Description | Key Inputs | Outputs |
|---|---|---|---|
| `find_object` | Look up runtime object | flow, string (object ID) | flow (success), flow (failure), object |
| `save_global` | Store value in global context | flow, string (key), object (value) | flow |
| `load_global` | Load value from global context | flow, string (key) | flow (success), flow (failure), object (value) |

### Value Literals (No Flow Pins)

These nodes produce a single output value with no flow connections:

| Type ID | Output Type | Default |
|---|---|---|
| `string` | string | `""` |
| `int` | int | `0` |
| `float` | float | `0.0` |
| `bool` | bool | `false` |
| `vector2` | vector2 | `{x: 0.0, y: 0.0}` |
| `color` | color | `{r: 0.0, g: 0.0, b: 0.0, a: 1.0}` |

### Asset References (No Flow Pins)

These nodes hold asset ID strings with a single output pin:

| Type ID | Output Type |
|---|---|
| `font` | font |
| `audio` | audio |
| `video` | video |
| `shader` | shader |
| `texture` | texture |

### Math and Logic

| Type ID | Description | Inputs | Output |
|---|---|---|---|
| `add` | Addition | object, object | object |
| `subtract` | Subtraction | object, object | object |
| `multiply` | Multiplication | object, object | object |
| `divide` | Division | object, object | object |
| `modulo` | Modulo | object, object | object |
| `greater` | Greater than comparison | object, object | bool |
| `equal` | Equality comparison | object, object | bool |
| `not` | Boolean NOT | bool | bool |
| `ceil` | Ceiling function | float | int |
| `floor` | Floor function | float | int |
| `round` | Round function | float | int |
| `merge_flow` | Merge up to 3 flow paths | flow x3 | flow |

### Utility

| Type ID | Description |
|---|---|
| `comment` | Annotation node (not executed). Has extra `text` and `size` fields. |
| `print` | Debug print to console |
| `string_concat` | Concatenate two strings |
| `random_int` | Generate random integer in range |

## How to Edit Generated Flow Files

### Finding a Specific Scene

Each scene in the assembled project has its own `.flow` file in `application/flow/`. The filename matches the scene ID (e.g., `ch_001_sc_001.flow`). The entry point of the project is `entry.flow`.

### Editing Dialogue Text

To change dialogue text, find the `show_dialog_box` node and edit the `val` field of its string pins:

```json
{
  "type_id": "show_dialog_box",
  "input_pin_list": [
    { "type_id": "flow", "is_output": false },
    { "type_id": "string", "is_output": false, "val": "Elena", "name": "..." },
    { "type_id": "string", "is_output": false, "val": "I never expected to find this here.", "name": "..." },
    ...
  ]
}
```

The first string pin (index 1) is the character name. The second string pin (index 2) is the dialogue content.

### Adding a New Choice

To add a branching choice, insert a `show_choice_button` node. Set the choice text in the string input pins (indices 1-5). Unused choices should have empty strings. Connect each output flow pin to the appropriate branch path.

### Changing Backgrounds

Find the `switch_background` node and update the `texture` pin's `val` to a different asset ID. The asset ID should match a filename (without extension) in your `application/resources/backgrounds/` directory.

### Validating Edits

After editing a `.flow` file, verify the following:

1. `max_uid` is at least as large as the highest ID in the file
2. All `id` values are unique across nodes, pins, and links
3. All `link_pool` entries reference existing pin IDs
4. Every flow has exactly one `entry` node

You can also use `vne validate yourfile.flow` to check for common issues.

## Common Patterns

### Dialogue Sequence

A basic dialogue exchange: entry -> dialogue -> wait -> dialogue -> wait.

```
entry -> show_dialog_box("Elena", "Hello.") -> show_dialog_box("Marcus", "Hi there.")
```

Each `show_dialog_box` node has a `wait` bool input (default `true`) that pauses for the player to click before continuing.

### Branching Choice

A two-option choice with convergence:

```
entry -> show_choice_button("Option A", "Option B", "", "", "")
           |                    |
           v                    v
     save_global("chose_a")  save_global("chose_b")
           |                    |
           v                    v
         merge_flow <-----------+
           |
           v
       (continue)
```

The `save_global` nodes record the player's choice for later reference. The `merge_flow` node reunites the branches.

### Scene Transition

To transition between scenes:

```
transition_fade_out(1.0) -> switch_scene("ch_001_sc_002")
```

The `switch_scene` node takes a string pin containing the target scene ID (the `.flow` filename without extension).

### Cinematic Narration

Letterboxed narration sequence:

```
show_letterboxing(200, 1.5) -> show_subtitle("The dust settled...") -> hide_subtitle -> hide_letterboxing(1.5)
```
